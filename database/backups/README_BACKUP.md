## 📋 Съдържание

Тази директория съдържа процедури и скриптове за управление на резервни копия на базата данни `amazon_brand_analytics`.

---

## 🔧 Налични процедури

### 1️⃣ `backup_table(table_name)`

**Описание:** Създава резервно копие на конкретна таблица с времеви печат (timestamp).

**Пример:**
```sql
CALL backup_table('products');
-- Резултат: products_backup_20260115_143025
```

---

### 2️⃣ `cleanup_old_backups(days_to_keep)`

**Описание:** Изтрива backup таблици по-стари от определен брой дни.

**Пример:**
```sql
CALL cleanup_old_backups(30);
-- Изтрива всички backup таблици по-стари от 30 дни
```

---

### 3️⃣ `table_size_stats()`

**Описание:** Показва статистика за размери на всички таблици в базата данни.

**Пример:**
```sql
CALL table_size_stats();
-- Връща: Име на таблица | Размер (MB) | Брой редове | Данни (MB) | Индекси (MB)
```

## 📅 График за backup

| Тип backup | Честота | Запазване |
|------------|---------|-----------|
| **Пълен backup** | Седмично (неделя, 02:00) | 4 седмици |
| **Инкрементален backup** | Дневно (всеки ден, 03:00) | 7 дни |
| **Таблици с висок приоритет** | 2 пъти дневно | 14 дни |

---

## 📌 Важни бележки

1. **Автоматизация:** Използвайте `cron` job или MySQL Event Scheduler за автоматично изпълнение на backup процедурите.
2. **Съхранение:** Препоръчва се backup файловете да се съхраняват на отделен сървър или облачно хранилище (AWS S3, Google Cloud Storage).
3. **Тестване:** Редовно тествайте възстановяването от backup, за да сте сигурни, че процесът работи коректно.

---

## 🚀 Бързи команди

```bash
# Експорт на цялата база данни (от командния ред)
mysqldump -u root -p amazon_brand_analytics > backup_full_$(date +%Y%m%d).sql

# Възстановяване от backup
mysql -u root -p amazon_brand_analytics < backup_full_20260115.sql

# Експорт само на структурата (без данни)
mysqldump -u root -p --no-data amazon_brand_analytics > schema_only.sql
```
