# Софтуер за Анализ на Брандове в Amazon

## Информация за Проекта

- **Текуща дата**: 19 Февруари 2026

## Обща Информация

Този софтуер предоставя изчерпателен анализ и наблюдение в реално време на представянето на брандове в платформата на Amazon. Позволява на продавачите да вземат решения базирани на данни за оптимизация на брандовете, рекламните кампании и SEO подобренията.

## Ключови Функционалности

1. Извличане на данни в реално време от платформата на Amazon
2. Проследяване на видимостта на брандовете в резултатите от търсене
3. Анализ на ключови думи и обем на търсене
4. Анализ на конкуренцията и бенчмаркинг
5. Препоръки за оптимизация на рекламни кампании
6. Предложения за SEO оптимизация
7. Показатели за представяне и анализ на тенденции

## Технологичен Стек

- **Backend**: Python 3.9+, Flask
- **Frontend**: JavaScript, HTML5, CSS3
- **База данни**: MySQL 8.0
- **API интеграции**: Amazon Product Advertising API, Amazon Selling Partner API
- **Обработка на данни**: Pandas, NumPy
- **Визуализация**: Chart.js

## Инсталация

### Необходими Изисквания
- Python 3.9 или по-висока версия
- MySQL 8.0 или по-висока версия
- Node.js 14+ (за frontend разработка)

### Инструкции за Настройка

1. Клонирайте repository:
```bash
git clone https://github.com/martinDachev/amazon-brand-analytics.git
cd amazon-brand-analytics
```

2. Инсталирайте Python зависимости:
```bash
pip install -r requirements.txt
```

3. Конфигурирайте базата данни:
```bash
mysql -u root -p < database/schemas/initial_schema.sql
```

4. Настройте environment променливи:
```bash
cp config/.env.example config/.env
# Редактирайте config/.env с вашите данни
```

5. Стартирайте приложението:
```bash
python backend/app.py
```

## API Документация

### Endpoints

- `GET /api/brands` - Списък на брандове с pagination
- `POST /api/brands` - Създаване на нов бранд
- `GET /api/brands/{id}/visibility` - Visibility score
- `GET /api/products/search?keyword=` - Търсене на продукти
- `GET /api/health` - Health check

## Техническа Архитектура

Системата е изградена на базата на microservices архитектура с:
- Flask REST API backend
- Redis за кеширане
- Celery за background jobs
- MySQL за data storage

## Deployment

```bash
# Production deployment
docker-compose up -d

# Database migration
flask db upgrade

# Start Celery worker
celery -A backend.app worker --loglevel=info
```
